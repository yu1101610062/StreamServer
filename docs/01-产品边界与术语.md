# 01. 产品边界与术语

## 1. 文档目标

本文件锁定 StreamServer 的 V1 范围、非目标、角色定义、任务类型和核心术语。实现、测试、联调、验收都以本文件为业务边界来源。

## 2. V1 目标

StreamServer V1 的目标是提供一个统一控制面，编排 ZLMediaKit 和 FFmpeg，在单一受信网络域内完成以下能力：

- 实时拉流转发与协议分发。
- 文件离线转码与产物归档。
- 文件转直播推流。
- UDP 组播桥接与 RTSP 组播场景支持。
- RTP/GB28181 接收。
- 统一任务模型、模板、默认值、调度、恢复、日志、审计和管理台。

## 3. V1 非目标

以下能力不属于 V1 正式承诺范围：

- 用户直接提交原始 FFmpeg 命令。
- 依赖 ZLMediaKit 商业版能力的转码或 IPTV 特性。
- 跨地域、多机房调度和异地容灾。
- 对公网直接暴露 ZLM 原始 HTTP API。
- 面向普通终端用户的消费级播放站点或 Portal。
- 完整对象存储归档与 SRT 深度能力。

## 4. 角色定义

| 角色 | 职责 | 权限边界 |
| --- | --- | --- |
| 平台管理员 | 管理节点、模板、系统配置、调试接口 | 拥有全部读写权限 |
| 业务调用方 | 创建、启动、停止、查询任务 | 仅能访问所属租户资源 |
| 观测/审计用户 | 只读查看任务、日志、录像、事件 | 只读，无调试和控制能力 |
| `media-core` | 控制面与真相库写入者 | 不直接处理媒体数据 |
| `media-agent` | 节点执行器 | 只能执行结构化任务计划 |
| ZLMediaKit | 实时媒体数据面 | 不承载业务权限和租户逻辑 |

## 5. V1 任务类型

| 类型 | 说明 | 典型执行器 | 输入 | 输出 |
| --- | --- | --- | --- | --- |
| `stream_ingest` | 把实时源或离线源接入为平台内部流，可选暴露播放协议、可选录制 | ZLM / FFmpeg + ZLM | RTSP、RTMP、HLS、HTTP-FLV、HTTP-TS、HTTP-MP4、文件、组播、GB-RTP | 内部流、分发协议、录像 |
| `stream_bridge` | 把源桥接到显式外部目标，只负责文件或组播输出 | FFmpeg | RTSP、RTMP、HLS、HTTP-FLV、HTTP-TS、HTTP-MP4、文件、组播 | 文件、UDP MPEG-TS 组播、RTP 组播 |
| `file_transcode` | 离线文件/VOD 转码、转封装、滤镜处理，只产出文件 | FFmpeg | 文件、HTTP-MP4、VOD HLS、VOD HTTP-TS | 文件 |

## 6. 能力边界

### 6.1 正式承诺

- 任务粒度的创建、启动、停止、取消、重试、克隆。
- `requested_spec` 与 `resolved_spec` 双份规格持久化。
- 基于 PostgreSQL 的状态机、事件流、租约和恢复。
- ZLM Hook 接入、标准化、落库和幂等处理。
- FFmpeg 能力探测、计划执行、日志与进度采集。
- Web 管理台 6 个基础页面。

### 6.2 条件支持

以下能力只在节点实际能力满足时启用：

- 某一编解码器或协议是否可用，以 `ffmpeg -protocols/-formats/-encoders/-decoders` 探测结果为准。
- 某一 ZLM 接口是否可用，以节点实际 `getApiList` 为准。
- H.265/Opus over RTMP 等高级能力默认关闭，仅在模板明确声明时启用。

### 6.3 明确排除

- 多租户资源配额系统。
- 自动财务计费、账单、套餐。
- 复杂审批流。
- 图形化流程编排。
- 跨集群统一调度。

## 7. 支持矩阵口径

| 能力 | V1 口径 |
| --- | --- |
| ZLM 开源版 RTSP/RTMP/HLS/HTTP-FLV/HTTP-TS/fMP4/MP4/GB28181/RTP | 正式支持 |
| RTSP `rtp multicast` | 正式支持 |
| 原始 UDP 组播输入/输出 | 通过 FFmpeg 正式支持 |
| 离线转码 | 正式支持 |
| 内部流接入 | 正式支持 |
| 流桥接 | 正式支持 |
| 多节点调度 | 设计纳入，首批实现可先单节点闭环 |
| GPU 硬编 | 预留，非正式承诺 |
| 对象存储归档 | 预留，非正式承诺 |

## 8. 核心术语

| 术语 | 定义 |
| --- | --- |
| Task | 上层唯一操作对象，代表一条可调度、可恢复、可审计的媒体工作单元 |
| Requested Spec | 用户原始提交的任务参数 |
| Resolved Spec | 合并系统默认值和用户请求后的最终执行参数 |
| Attempt | 任务的一次具体执行尝试 |
| Lease | 任务在某节点上的排他执行租约 |
| Binding | 任务与底层流、ZLM 对象、RTP 端口、推流器之间的绑定关系 |
| Checkpoint | 可恢复任务在执行过程中的持久化进度点 |
| Node Capability | 节点探测到的 FFmpeg、ZLM、网络、GPU 等能力集合 |
| Hook Event | ZLM 通过 Web Hook 发来的运行时事件 |

## 9. 默认业务约束

- 上层只允许操作任务，不允许直接调用 ZLM API。
- 所有可变更写接口必须支持幂等键。
- 所有任务必须能追溯到租户、操作人、模板、节点和执行尝试。
- 默认采用内网受信部署，不为公网开放裸媒体控制接口。
