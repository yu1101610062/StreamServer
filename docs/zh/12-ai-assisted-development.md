# 12. AI 辅助开发说明

StreamServer 的部分需求梳理、代码草稿、文档整理、测试补充、错误日志分析和重构方案会借助 AI 工具完成。AI 工具在项目中承担辅助角色，用于提高信息整理、实现探索和验证准备的效率。

仓库中的最终代码、文档和脚本以实际提交内容为准。AI 生成内容会经过项目维护者筛选、修改、审查和验证后再进入代码库。

## 使用方式

项目开发过程中，AI 工具主要参与以下辅助工作：

- 整理需求、接口边界和任务拆分。
- 生成代码草稿、测试草稿和文档草稿。
- 分析错误日志、测试失败和部署问题。
- 辅助整理 README、ADR、变更说明和优化报告。
- 辅助检查文档与实现之间的一致性。

这些工作不改变项目维护方式。项目维护者仍负责需求判断、架构设计、代码审查、测试验收、部署验证和后续维护。

## 对外说明

StreamServer is developed with AI-assisted engineering workflows. Project maintainers remain responsible for requirements, architecture, code review, testing, deployment validation, and iterative refactoring.
