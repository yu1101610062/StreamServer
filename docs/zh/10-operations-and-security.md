# 10. 运维、安全与风险预案

## 1. 文档目标

本文件用于提前记录开发和联调阶段的主要风险、触发条件、影响、监测信号和预案。风险项在进入实现前必须被团队知晓。

## 2. 风险清单

| 风险 | 触发条件 | 影响 | 监测信号 | 预案 |
| --- | --- | --- | --- | --- |
| FFmpeg build 缺能力 | 节点未编入某协议或编码器 | 任务无法启动 | `capability_snapshot` 缺字段 | 启动前做能力校验，不满足直接拒绝 |
| ZLM Hook 重复或乱序 | 网络重试、节点重连 | 重复入库或状态回退 | 同一 `dedup_key` 重复出现 | 原始 Hook 落库并去重 |
| 组播网络不可用 | 交换机、路由、防火墙配置不对 | `multicast_bridge` 无法工作 | 组播加入失败、收不到包 | 开工前先做宿主机网卡和路由验证 |
| ZLM 重启后流丢失 | 服务重启、崩溃 | 运行任务失联 | `on_server_started` | 触发恢复引擎重新对账或重建 |
| Agent 与 Core 断连 | 网络闪断或 Core 重启 | 心跳中断、控制失效 | 节点心跳超时 | 采用长连接重试和 orphan 接管 |
| 录制目录不可写 | 权限、磁盘满 | 录像失败或任务异常 | `record_file_created` 缺失、磁盘告警 | 启动前检测目录和剩余空间 |
| 状态机漂移 | 代码绕过统一状态转换 | 数据不一致 | 非法状态组合 | 强制通过领域服务变更状态并加测试 |
| 规格与实现不一致 | 文档没同步更新 | 联调误解、返工 | 文档字段缺失 | 接口和表结构变更必须先更新文档 |
| 时钟漂移 | 节点时间不一致 | 事件顺序混乱 | 时间戳异常跳变 | NTP 同步，事件排序优先用数据库时间 |
| 录制不计为 viewer | ZLM 默认语义 | 无观众自动关流误伤录制任务 | `stream_no_reader` 频繁触发 | 默认 `auto_close=0`，录制视为保活条件 |

## 3. 开发期重点关注

- 先验证单节点闭环，再扩大到多节点和组播。
- 任何“可能由商业版能力覆盖”的功能都先用开源版实际能力验证。
- 先做能力探测和显式失败，再追求自动兼容。

## 4. 上线前复核

- 组播网络已实测。
- ZLM Hook 已验证白名单和 shared secret。
- 节点能力快照已成功写库。
- 恢复场景至少验证 Core、Agent、ZLM 三类重启。
